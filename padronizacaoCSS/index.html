Padronização CSS

O que é o CSS?

Enquanto o HTML é a estrutura lógica da página, o CSS é o design e a beleza.
Imagine que na construção de um prédio, o HTML é o engenheiro e o CSS o arquiteto.

A estrutura HTML dessa página e da de hierarquia é quase a mesma. O que muda principalmente é a presença do CSS.

Inline vs arquivo

Podemos estilizar o HTML de 2 maneiras.

  • Inline
    • Usando atributo style direto na tag.
      • Exemplo a cima: <li style="color: red; font-weight: 800;">Usando ...</li>
  • Arquivo css
    • Podemos criar arquivos separados do HTML contendo nossa estilização.
    • Esses arquivos devem ser adicionados na tag <head> do HTML onde serão utilizados.

Organização - Tag vs ID vs Classe

Podemos selecionar quais itens serão estilizados de diversas maneiras. As 3 mais comuns são:

  • Por Tag
    • No arquivo CSS, podemos fazer uma estilização para todas as tags do tipo selecionado contidas naquela página.
      • Exemplo: h1{font-size: 2.5rem;}
  • Por ID
    • Podemos selecionar uma tag específica, marcada por uma id (id="nomeDaId") e acessa-la no documento CSS usando #
      • Exemplo: #nomeDaId{color: blue;}
  • Por Classe
    • Podemos selecionar um grupo de tags, marcadas por uma classe (class="nomeDaClasse") e acessa-las no documento CSS usando.
      • Exemplo: .nomeDaClasse{color: purple;}

Padronizando seu código

Existem algumas dicas padrão para organização e padronização de código.

  • Crie um documento de estilização global.
    • Nele, você fará a organização básica da página. Padding das laterais, espaçamento do texto, fonte, tamanho da fonte dos de títulos e parágrafos, etc.
  • Faça vários arquivos de CSS
    • O tempo passa, você faz cada vez mais projetos e a organização para manutenção do CSS é muito importante. Crie arquivos separados para header, footer, sections, etc. Você vai agradecer por não ter que prestar manutenção em páginas com 500 linhas de css.
    • Eu mesmo gostaria de ter aprendido isso antes kkk.
  • Faça muitos comentários pelo seru código.
    • Agora não parece útil, mas com o crescimento do projeto e das suas habilidades, é um hábito muito importante.
